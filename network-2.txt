포트 Port
네트워크를 통해 데이터를 주고받는 프로세스를 식별하기 위해 호스트 내부적으로
프로세스가 할당받는 고유 값
IP address 내에 개별적으로 부여된 통신 프로세스
숫자로 표현되므로 포트 넘버라고도 함.
1~1024까지는 시스템이 사용함.

소켓 Socket
두 시스템 사이의 네트워크 연결을 나타내는 객체
소켓을 열기 위해서는 호스트 IP, 포트넘버, 프로토콜이 필요하다.
보내는 쪽 / 받는 쪽 소켓을 둘다 열어야하고, 프로세스가 같은 포트를
가지고 여러개의 소켓을 열 수 있다.

Subnetmask => class로 분리하던 IP 주소 체계에서, 네트워크/호스트 부분으로
IP 영역대를 분리한 것
특정 IP와 비트수를 이용하여 네트워크 영역을 구분하고,
Longest prefix match 를 많이 사용하니까 기억하기

TCP와 UDP의 차이
TCP - 연결형 서비스를 지원.
연결 수립할 때 3 hand shaking, 끝낼 떄 4 hand shaking 
연속성보다 신뢰성있는 전송을 생각할 때 TCP를 사용

UDP - 비연결형 서비스를 지원
전송 순서를 보장하지 않고, 신뢰성이 낮지만 속도가 빠름. ex) 스트리밍, 음성, 멀티미디어
헤더의 크기가 8바이트로 TCP에 비해 매우 작다 .


TCP / UDP => TCP의 통신 방식은 1:1, UDP는 1:1 , 1:N, N:N 가능

HTTP 
WWW 상에서 정보를 주고 받을 수 있는 프로토콜
HTML 문서를 주고 받는데에 사용하게 된다.
80번 포트를 사용하고, HTTPS가 나타난 이유는 HTTP의 보안상 문제가 많았고
HTTP에 SSL, TLS를 이용하여 암호화하여 주고 받음 port 443
보안을 위해 사용하는 세션키를 생성 / 교환하는 과정이 필요하고
이를 SSL HandShake라고 함.
