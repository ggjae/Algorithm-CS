서버와 클라이언트 => 클라이언트가 요청을 보내면 서버가 응답합니다.

서버 => 서비스 제공자

HTTP : 웹 상에서 데이터를 주고 받기 위한 프로토콜
GET -> 특정한 데이터의 조회를 요청 ) 특정 페이지 접속, 정보 검색
POST -> 특정한 데이터의 생성을 요청 ) 회원가입, 글쓰기
PUT -> 특정한 데이터의 수정을 요청 ) 회원 정보 수정
DELETE -> 특정한 데이터의 삭제를 요청 ) 회원 정보 삭제

REST API, JSON
REST -> 다양한 HTTP 메소드가 있는데, 사용자 생성할 때 DELETE 메서드를
받으면 안될까..? => 기본 개념적으론 안되잖아.
